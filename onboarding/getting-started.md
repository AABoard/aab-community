# Getting Started with AAB Collaboration

Welcome to the AAB community collaboration space. This guide helps you find the right place to contribute.

## Step 1: Choose Your Contribution Area

- Case or pilot evidence: start with `aab-data-and-schema`.
- Schema, export, search, or validation work: start with `aab-data-and-schema`.
- Standards comments or draft language: start with `aab-standards`.
- Governance policy or role questions: start with `aab-governance`.
- Unsure where to help: open a contributor intake issue here.

## Step 2: Prepare Public, Reviewable Material

Good contributions include source links, context, the affected audience, and a clear question or proposed change. Avoid private student data, credentials, confidential partner records, and unpublished pilot details.

## Step 3: Use GitHub Issues and Pull Requests

Use issues for discussion, intake, and review questions. Use pull requests when proposing concrete edits to documentation, schemas, templates, or standards drafts.

## Step 4: Expect Review

AAB review is designed to protect neutrality, privacy, source traceability, and public trust. Registry inclusion, standards language, and governance updates require review before being treated as official AAB output.
