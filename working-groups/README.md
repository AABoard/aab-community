# Working Groups

AAB working groups organize contributor activity around clear public workflows.

## Current Working Group Areas

| Working group | Focus |
| --- | --- |
| Case review | Source-traced case and pilot documentation. |
| Data and schema | Schemas, templates, validation, exports, and search. |
| Standards | Standards drafts, comments, evidence trace, and grade-band progression. |
| Policy monitoring | Public policy and community-signal tracking. |
| Research synthesis | Research briefs, framework crosswalks, and evidence summaries. |
| Website and tools | Public website, registry UX, and contributor tooling. |
| Community dialogue | Contributor onboarding, events, and discussion prompts. |

## How To Join

Open a contributor intake issue and identify the working group you want to support. If you are unsure, describe your background and interests and a maintainer will route you.

## Working Group Notes

Public notes should avoid private student data, confidential partner information, unpublished pilot details, credentials, and private contact lists.
